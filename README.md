# 起動方法
`docker-compose up --build`を実行してサービスを起動します.
`http://localhost:5173`にアクセスすることでサービスを使用することができます.

# Web公開
GitHub Pages で公開する場合は、`main` ブランチへ push すると `.github/workflows/deploy.yml` が実行され、静的ファイルが自動デプロイされます.

公開先は GitHub Pages の設定画面で確認できます. GitHub Pages 上ではデモデータを使うため、一覧表示や登録操作をそのまま試せます. 実データで運用する場合は、`src/server.js` の API を別途公開して接続してください.


# サービスの機能

## マスタ管理
左のサイドメニューの「マスタ」から各マスタの管理画面に遷移することができます.右上の2つのボタンによって,「閲覧」,「編集」モードを切り替えることができます.編集モードでは,マスタの新規登録,編集,削除を行うことができます.
閲覧モードではそれらの操作を行うことはできません.

## 構成
左のサイドメニューの「構成」から構成ツリーの表示画面に遷移することができます.完成品を選択することでその下に,その完成品を構成する部品の一覧が表示されます.その中で中間品を選択すると中間品を構成する部品の一覧が表示されます.このようにして,完成品を構成する部品の階層構造を確認することができます.

## 入荷
左のサイドメニューの「入荷」から入荷画面に遷移することができます.部品を選択し,入荷数を入力することでその部品の在庫数を増やすことができます.また,入荷履歴も確認することができます.

## 出荷
左のサイドメニューの「出荷」から出荷画面に遷移することができます.部品を選択し,出荷数を入力することでその部品の在庫数を減らすことができます.また,出荷履歴も確認することができます.

## 組立
左のサイドメニューの「組立」から組立画面に遷移することができます.完成品を選択し,組立数を入力することでその完成品を構成する部品の在庫数を減らすことができます.また,組立履歴も確認することができます.在庫数が足りない場合は組立を行うことができません.

## 在庫
左のサイドメニューの「在庫」から在庫画面に遷移することができます.部品の在庫数を確認することができます.
